85     # +has_attached_file+ gives the class it is called on an attribute that maps to a file. This
 
86     # is typically a file stored somewhere on the filesystem and has been uploaded by a user. 
87     # The attribute returns a Paperclip::Attachment object which handles the management of
 
88     # that file. The intent is to make the attachment as much like a normal attribute. The 
89     # thumbnails will be created when the new file is assigned, but they will *not* be saved 
90     # until +save+ is called on the record. Likewise, if the attribute is set to +nil+ is 
91     # called on it, the attachment will *not* be deleted until +save+ is called. See the 
92     # Paperclip::Attachment documentation for more specifics. There are a number of options 
93     # you can set to change the behavior of a Paperclip attachment:
 
94     # * +url+: The full URL of where the attachment is publically accessible. This can just
 
95     #   as easily point to a directory served directly through Apache as it can to an action
 
96     #   that can control permissions. You can specify the full domain and path, but usually
 
97     #   just an absolute path is sufficient. The leading slash must be included manually for 
98     #   absolute paths. The default value is "/:class/:attachment/:id/:style_:filename". See
 
99     #   Paperclip::Attachment#interpolate for more information on variable interpolaton.
 
100     #     :url => "/:attachment/:id/:style_:basename:extension"
 
101     #     :url => "http://some.other.host/stuff/:class/:id_:extension"
 
102     # * +default_url+: The URL that will be returned if there is no attachment assigned. 
103     #   This field is interpolated just as the url is. The default value is 
104     #   "/:class/:attachment/missing_:style.png"
 
105     #     has_attached_file :avatar, :default_url => "/images/default_:style_avatar.png"
 
106     #     User.new.avatar_url(:small) # => "/images/default_small_avatar.png"
 
107     # * +styles+: A hash of thumbnail styles and their geometries. You can find more about 
108     #   geometry strings at the ImageMagick website 
109     #   (http://www.imagemagick.org/script/command-line-options.php#resize). Paperclip
 
110     #   also adds the "#" option (e.g. "50x50#"), which will resize the image to fit maximally 
111     #   inside the dimensions and then crop the rest off (weighted at the center). The 
112     #   default value is to generate no thumbnails.
 
113     # * +default_style+: The thumbnail style that will be used by default URLs. 
114     #   Defaults to +original+.
 
115     #     has_attached_file :avatar, :styles => { :normal => "100x100#" },
 
116     #                       :default_style => :normal
 
117     #     user.avatar.url # => "/avatars/23/normal_me.png"
 
118     # * +whiny_thumbnails+: Will raise an error if Paperclip cannot process thumbnails of an
 
119     #   uploaded image. This will ovrride the global setting for this attachment. 
120     #   Defaults to true. 
121     # * +convert_options+: When creating thumbnails, use this free-form options
 
122     #   field to pass in various convert command options.  Typical options are "-strip" to
 
123     #   remove all Exif data from the image (save space for thumbnails and avatars) or
 
124     #   "-depth 8" to specify the bit depth of the resulting conversion.  See ImageMagick
 
125     #   convert documentation for more options: (http://www.imagemagick.org/script/convert.php)
 
126     #   Note that this option takes a hash of options, each of which correspond to the style
 
127     #   of thumbnail being generated. You can also specify :all as a key, which will apply
 
128     #   to all of the thumbnails being generated. If you specify options for the :original,
 
129     #   it would be best if you did not specify destructive options, as the intent of keeping
 
130     #   the original around is to regenerate all the thumbnails then requirements change.
 
131     #     has_attached_file :avatar, :styles => { :large => "300x300", :negative => "100x100" }
 
132     #                                :convert_options => {
 
133     #                                  :all => "-strip",
 
134     #                                  :negative => "-negate"
 
135     #                                }
